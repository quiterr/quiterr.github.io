# 线程池

多线程编程要小心，必须要加try catch，否则可能出现线程挂掉，连异常日志都没有。

阿里的Java规范建议手动创建线程池：

ExecutorService pool = new ThreadPoolExecutor(1, 4,
        1000, TimeUnit.MILLISECONDS, new LinkedBlockingDeque<>(2000),
        Executors.defaultThreadFactory(),new ThreadPoolExecutor.AbortPolicy());

创建定时任务线程池：

ScheduledExecutorService scheduledThreadPool = Executors.newScheduledThreadPool(5);

如果跟进去看的话，第二个最大线程数设的是Integer.MAX_VALUE，可能会耗尽系统计算资源：

public ScheduledThreadPoolExecutor(int corePoolSize) {
    super(corePoolSize, Integer.MAX_VALUE, 0, NANOSECONDS,
            new DelayedWorkQueue());
}

手动创建定时任务线程池：

ScheduledThreadPoolExecutor ex = new ScheduledThreadPoolExecutor(5, 
        Executors.defaultThreadFactory(), 
        new ThreadPoolExecutor.AbortPolicy());

参考资料：https://www.cnblogs.com/dafanjoy/p/9729358.html

# 函数式编程

idea会提示：

  • 能用lambda却用了匿名类
  • 能用方法引用却用了lambda

lambda

TimeTrigger timeTrigger = new TimeTrigger(10000L, 10000L);
DataTrigger dataTrigger = new DataTrigger();

List<AbstractTrigger> triggers = new ArrayList<>();
triggers.add(timeTrigger);
triggers.add(dataTrigger);

List<TimeTrigger> timeTriggers = triggers.stream()
        .filter((AbstractTrigger trigger) ->trigger instanceof TimeTrigger)
        .map((AbstractTrigger trigger) -> (TimeTrigger) trigger)
        .collect(Collectors.toList());

方法引用

timeTriggers.forEach(System.out::println);
