# 问题集锦

这篇文档记录项目实践中遇到的一些技术问题。

# 布局问题

原型图如下

这个简单的页面是我第一个实际项目的第一个页面，做了四次才达标。

第一版用的absolute定位，但是最外层的container没有高度，因为absolute是浮起来的，不占DOM文档流，这样没法给container设置height:auto。

第二版用的relative定位，但是安全设置的div右侧超出页面了，不像absolute时用left、right可以限制住。

第三版用的padding，好多了，但是“企业简介”如果字数很多或者窗口变窄，都会导致换行，换行后对齐到“企业简介”的“企业”那里去了。还有安全设置的“登录密码”的第二列也是一样，窗口变窄之后，中间列不会换行，而是第三列的按钮先换行。

第四版就用了flex来解决上述的问题。

# 两边固定，中间自适应

有问题的效果是用左边span设置display:inline-block固定宽度并居中，中间span用的margin-left，右边span用的float和relative。当窗口变窄之后，“修改密码”按钮就换行了。

改用flex，每一行的div设置为

.item-div {
    display: flex;
    align-items: center;
    justify-content: center;
    height: 3.69rem;
    border-bottom: 1px solid rgba(230, 230, 230, 1);
    font-size: 0.875rem;
}

三列设置为

 .item-col-1 {
    flex: 0 0 8rem; /*不放大、不缩写、初始大小*/
    text-align: center;
}

.item-col-2 {
    line-height: 1.5em;
    color: #999999;
}

.item-col-3 {
    margin-left: auto;
    flex: 0 0 8rem;
    text-align: center;
    color: #5187ED;
}

# 首页-登录页

1、登录页面，返回首页的a标签为啥不能调整字体大小? 由于reset.css中定义了a标签的字体大小为100%，就把自己定义的a标签的样式覆盖了。 解决办法：一个是给a标签定义一个id或class，或者定义样式为font-size: 16px !important;

2、登录页面，el-main怎么会有20px的padding，用/deep/又把高度整没了 20px的padding是组件自带的样式，/deep/是网上查到的用法，但 /deep/ .el-main这样是没用的，正确用法：.el-container /deep/ .el-main

3、登录页面，第一次实现的时候没有用响应式，页面缩小之后就变形了，加了element-ui的响应式栅格后，logo图片却不能垂直居中对齐。 解决方法：行内元素垂直居中vertical-align: middle;

# 顶部导航栏下方加阴影

CSS代码

box-shadow: 0 1px 4px rgba(0, 21, 41, 0.08);

但实际并没有生效，看起来像是被下方的内容覆盖了，需要加两行代码

box-shadow: 0 1px 4px rgba(0, 21, 41, 0.08);
position: relative;
z-index: 1000;

加了position: relative;之后z-index: 1000;才会生效。
