# 1.1.3 v-for指令

直接贴代码

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <script type="text/javascript" src="../assets/js/vue.js"></script>
    <title>V-for 案例</title>
</head>

<body>
    <h1>v-for指令用法</h1>
    <hr>
    <div id="app">
        <ul>
            <li v-for="item in sortItems">
                {{item}}
            </li>
        </ul>
        <ul>
            <li v-for="student in sortStudent">
                {{student.name}} - {{student.age}}
            </li>
        </ul>
    </div>
    <script type="text/javascript">
        var app = new Vue({
            el: '#app',
            data: {
                items: [20, 23, 18, 65, 32, 19, 5, 56, 41],
                students: [
                    { name: 'jspang', age: 32 },
                    { name: 'Panda', age: 30 },
                    { name: 'PanPaN', age: 21 },
                    { name: 'King', age: 45 }
                ]
            },
            computed: {
                sortItems: function () {
                    return this.items.sort(sortNumber);
                },
                sortStudent: function () {
                    return sortByKey(this.students, 'age');
                }
            }
        });
        function sortNumber(a, b) {
            return a - b
        };
        //数组对象方法排序:
        function sortByKey(array, key) {
            return array.sort(function (a, b) {
                var x = a[key];
                var y = b[key];
                return ((x < y) ? -1 : ((x > y) ? 1 : 0));
            });
        };
    </script>
</body>

</html>

# 1.1.8 不渲染、渲染完后显示、仅渲染一次

v-pre指令

在模板中跳过vue的编译，直接输出原始值。

<div v-pre>{{message}}</div>

v-cloak指令

在vue渲染完指定的整个DOM后才进行显示。它必须和CSS样式一起使用

[v-cloak] {
  display: none;
}

<div v-cloak>
  {{ message }}
</div>

v-once指令

在第一次DOM时进行渲染，渲染完成后视为静态内容，跳出以后的渲染过程。

<div v-once>第一次绑定的值：{{message}}</div>
<div><input type="text" v-model="message"></div>

# 1.2 Vue.x api

vue提供了一些全局api来供我们调用，例如Vue.directive，Vue.extend等，本节学习这些api的用法。

# 1.2.1 Vue.directive 自定义指令

比如我们要定义一个v-jspang的指令，作用就是让文字变成红色。

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <script type="text/javascript" src="../../assets/js/vue.js"></script>
    <title>v-directive</title>
</head>

<body>
    <h1>v-directive</h1>
    <hr>
    <div id="app">
        <div v-jspang="color">{{number}}</div>
        <p><button @click="add">Add</button></p>
    </div>
    <script type="text/javascript">
         Vue.directive('jspang', function (el, binding, vnode) {
             console.log(el);
             console.log(binding);
             el.style = 'color:' + binding.value;
         });

        var app = new Vue({
            el: '#app',
            data: {
                number: 10,
                color: 'red'
            },
            methods: {
                add: function () {
                    this.number++;
                }
            }
        })
    </script>
</body>

</html>

自定义指令中传递的三个参数

  • el: 指令所绑定的元素，可以用来直接操作DOM。
  • binding: 一个对象，包含指令的很多信息。
  • vnode: Vue编译生成的虚拟节点。

自定义指令的生命周期

Vue.directive('jspang', {
	bind: function (el,binding) {//被绑定
		console.log('1 - bind');
		el.style = 'color:' + binding.value;
	},
	inserted: function () {//绑定到节点
		console.log('2 - inserted');
	},
	update: function () {//组件更新
		console.log('3 - update');
	},
	componentUpdated: function () {//组件更新完成
		console.log('4 - componentUpdated');
	},
	unbind: function () {//解绑
		console.log('5 - unbind');
	}
});

# 1.2.2 Vue.extend构造器的延伸

什么是Vue.extend？

Vue.extend 返回的是一个“扩展实例构造器”,也就是预设了部分选项的Vue实例构造器。经常服务于Vue.component用来生成组件，可以简单理解为当在模板中遇到该组件名称作为标签的自定义元素时，会自动调用“扩展实例构造器”来生产组件实例，并挂载到自定义元素上。

由于我们还没有学习Vue的自定义组件，所以我们先看跟组件无关的用途。

自定义无参数标签

我们想象一个需求：在博客页面多处显示作者的网名，并在网名上直接有链接地址。我们希望只需要在html中写 ，这和自定义组件（自定义组件还没学）很像，但是他没有传递任何参数，只是个静态标签。完整示例：

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <script type="text/javascript" src="../../assets/js/vue.js"></script>
    <title>v-extend</title>
</head>

<body>
    <h1>v-extend</h1>
    <hr>
    <author>
    </author>
    <script type="text/javascript">
        var authorExtend = Vue.extend({
            template: "<p><a :href='authorUrl'>{{authorName}}</a></p>",
            data: function () {
                return {
                    authorName: 'JSPang',
                    authorUrl: 'http://www.jspang.com'
                }
            }
        });
        new authorExtend().$mount('author');
    </script>
</body>

</html>

挂载到普通标签上

还可以通过HTML标签上的id或者class来生成扩展实例构造器，Vue.extend里的代码是一样的，只是在挂载的时候，我们用类似jquery的选择器的方法，来进行挂载就可以了。

new authorExtend().$mount('#author');

# 1.2.3 Vue.set全局操作

Vue.set 的作用就是在构造器外部操作构造器内部的数据、属性或者方法。比如在vue构造器内部定义了一个count为1的数据，我们在构造器外部定义了一个方法，要每次点击按钮给值加1.就需要用到Vue.set。

一、引用构造器外部数据

看一个简单的代码：

//在构造器外部声明数据
var outData={
    count:1,
    goodName:'car'
};
var app=new Vue({
    el:'#app',
    //引用外部数据
    data:outData
})

在外部改变数据的三种方法：

用Vue.set改变

function add(){
       Vue.set(outData,'count',4);
 }

用Vue对象的方法添加

app.count++;

直接操作外部数据

outData.count++;

二、为什么要有Vue.set的存在?

由于Javascript的限制，Vue不能自动检测以下变动的数组。

  • 当你利用索引直接设置一个项时，vue不会为我们自动更新。
  • 当你修改数组的长度时，vue不会为我们自动更新。

这个例子可以直接看讲义或代码。总之就是这样才能更新数组：

var outData = {
            count:1,
            // goods:'car'
            arr:['aaa','bbb','ccc']
        }
Vue.set(app.arr,1,'ddd');

# 1.2.6 组件

其实组件就是自定义的标签。

注意： 本节介绍的创建组件方式生产中很少用，一般都是单文件组件，例如App.vue这样的。

一、全局化注册组件

全局化就是在构造器的外部用Vue.component来注册

<body>
    <h1>component-1</h1>
    <hr>
    <div id="app">
        <jspang></jspang>
    </div>
    <script type="text/javascript">
        //注册全局组件
        Vue.component('jspang',{
            template:`<div style="color:red;">全局化注册的jspang标签</div>`
        })
        var app=new Vue({
            el:'#app',
            data:{
            }
        })
    </script>
</body>

二、局部注册组件局部

局部注册的组件只能在组件注册的作用域里进行使用，其他作用域使用无效。

<body>
    <h1>component-1</h1>
    <hr>
    <div id="app">
      <panda></panda>
    </div>
    <script type="text/javascript">
        var app=new Vue({
            el:'#app',
            components:{
                "panda":{
                    template:`<div style="color:red;">局部注册的panda标签</div>`
                }
            }
        })
    </script>
</body>

三、组件和指令的区别

组件注册的是一个标签，而指令注册的是已有标签里的一个属性。在实际开发中我们还是用组件比较多，指令用的比较少。因为指令看起来封装的没那么好，这只是个人观点。

# 1.2.7 props属性

props选项就是设置和获取标签上的属性值。

一、定义属性并获取属性值

<body>
    <h1>component-2</h1>
    <hr>
    <div id="app">
      <panda here="China"></panda>
    </div>
    <script type="text/javascript">
        var app=new Vue({
            el:'#app',
            components:{
                "panda":{
                    template:`<div style="color:red;">Panda from {{ here }}.</div>`,
                    props:['here']
                }
            }
        })
    </script>

# 2.2.8 路由的过渡动画

transition标签是vue支持的标签，非html原生。

想让路由有过渡动画，需要在<router-view>标签的外部添加<transition>标签，标签还需要一个name属性。

<transition name="fade">
  <router-view ></router-view>
</transition>

css过渡类名： 组件过渡过程中，会有四个CSS类名进行切换，这四个类名与transition的name属性有关，比如name=”fade”,会有如下四个CSS类名：

  1. fade-enter:进入过渡的开始状态，元素被插入时生效，只应用一帧后立刻删除。
  2. fade-enter-active:进入过渡的结束状态，元素被插入时就生效，在过渡过程完成后移除。
  3. fade-leave:离开过渡的开始状态，元素被删除时触发，只应用一帧后立刻删除。
  4. fade-leave-active:离开过渡的结束状态，元素被删除时生效，离开过渡完成后被删除。

从上面四个类名可以看出，fade-enter-active和fade-leave-active在整个进入或离开过程中都有效，所以CSS的transition属性在这两个类下进行设置。

那我们就在App.vue页面里加入四种CSS样式效果，并利用CSS3的transition属性控制动画的具体效果。代码如下：

.fade-enter {
  opacity:0;
}
.fade-leave{
  opacity:1;
}
.fade-enter-active{
  transition:opacity .5s;
}
.fade-leave-active{
  opacity:0;
  transition:opacity .5s;
}

transition标签还支持一个mode参数

<transition name="fade" mode="out-in">

# 2.3.4 getters计算过滤操作

getters从表面是获得的意思，可以把他看作在获取数据之前进行的一种再编辑,相当于对数据的一个过滤和加工。你可以把它看作store.js的计算属性。

比如每次获取count的时候加100，要在store.js中编写

const getters = {
    count: function (state) {
        return state.count += 100;
    }
}

// 个人理解：这里export后，xx.vue中才能用this.$store.getters.count;
export default new Vuex.Store({
    state,
    mutations,
    getters
})

在vue 的构造器里边只能有一个computed属性，如果你写多个，只有最后一个computed属性可用。在count.vue中编写代码，使用ES6中的展开运算符”…”

computed:{
    ...mapState(["count"]),
    count(){
        return this.$store.getters.count;
    }
},

用mapGetters简化模板写法

import { mapState,mapMutations,mapGetters } from 'vuex';

在computed中加入mapGetters

...mapGetters(["count"])

# 2.3.5 actions异步修改状态

在store.js中声明actions

const actions ={
    addAction(context){
        context.commit('add',10)
    },
    reduceAction({commit}){
        commit('reduce')
    }
}

在actions里写了两个方法addAction和reduceAction，在方法体里，我们都用commit调用了Mutations里边的方法。细心的小伙伴会发现这两个方法传递的参数也不一样。

  • context：上下文对象，这里你可以理解称store本身。
  • {commit}：直接把commit对象传递过来，可以让方法体逻辑和代码更清晰明了。

组件中使用

加两个按钮

<p>
  <button @click="addAction">+</button>
  <button @click="reduceAction">-</button>
</p>

引入mapActions

import { mapState, mapMutations, mapGetters, mapActions } from "vuex";

编写methods

methods:{
    ...mapMutations([  
        'add','reduce'
    ]),
    ...mapActions(['addAction','reduceAction'])
},

增加异步检验

现在看的效果和用Mutations作的一模一样，肯定有的小伙伴会好奇，那actions有什么用，为了演示actions的异步功能，在addAction里使用setTimeOut进行延迟执行。

setTimeOut(()=>{context.commit(reduce)},10000);
console.log('我比reduce提前执行');

注意：这里最好把getters的加100去掉，否则有点影响观察。

# 4.1.6.x 如何覆盖element-ui的样式？

这里有一个问题，就是第一个tab的padding-left被设置成了0，暂不清楚怎么回事。

“点餐”这个tab的padding-left被设置成了0，导致页面不是很美观，应该是element-ui自己的样式规则。

如何修改呢？第一反应就是在浏览器里边用F12的元素选择找到该tab的id或class，然后在css中修改：

.el-tabs--top.el-tabs__item.is-top:nth-child(2) {
    padding-left: 20px;
}

但是并没有生效，谷歌发现有人说是不能放在<style scoped>下，但即使放在<style>下也无效。

第二个想法是给“点餐”tab加上自定义的id或class，指定样式，但仍然没用，应该是被element的样式覆盖了。

其实只要F12选择元素的时候不看css，看html就会发现“点餐”tab的id是tab-0，最终方案是添加

<style>
#tab-0 {
    padding-left: 20px;
}
</style>
