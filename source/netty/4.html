# netty源码学习-多重继承、异步编程、观察者

这节主要学习NioEventLoopGroup，涉及知识点：多重继承（继承多个接口），可变长度参数列表，位运算替代取余运算提升性能；匿名类；观察者模式；异步编程。

# 类图

下面是NioEventLoopGroup的类图：

WARNING

从这个类图可以看出来，Java是支持接口的多重继承的。

图形上面部分的Executor、ExecutorService、ScheduledExecutorService、Iterable都是JDK定义的接口，其中前三个是java.util.concurrent包下的内容，相关内容可参考Java 并发工具包。

到这里可以明确NioEventLoopGroup是一个支持定时任务的线程池，而且内部一定有一个集合用来存放某种数据，支持对该集合进行迭代遍历。

一般我们要创建一个支持定时任务的线程池可以用Executors.newScheduledThreadPool(1);。多说一句，阿里的Java开发规范建议手动创建线程池，因为Executors.newFixedThreadPool(1);内部用的是无参数的构造函数来创建LinkedBlockingQueue，也就意味着任务队列可以很大，当任务处理慢时可能会OOM。

/**
* Creates a {@code LinkedBlockingQueue} with a capacity of
* {@link Integer#MAX_VALUE}.
*/
public LinkedBlockingQueue() {
    this(Integer.MAX_VALUE);
}

显然netty并没有用jdk的线程池，而是自己实现了一套。AbstractEventExecutorGroup中定义了submit、schedule、scheduleAtFixedRate、shutdownGracefully、invokeAll、invokeAny、execute等方法。

MultithreadEventExecutorGroup似乎是定义了一些业务相关的方法和变量:

private final EventExecutor[] children;
private final Set<EventExecutor> readonlyChildren;
private final AtomicInteger terminatedChildren = new AtomicInteger();
private final Promise<?> terminationFuture = new DefaultPromise(GlobalEventExecutor.INSTANCE);
private final EventExecutorChooserFactory.EventExecutorChooser chooser;

MultithreadEventLoopGroup代码要少一些，主要是DEFAULT_EVENT_LOOP_THREADS，还有newDefaultThreadFactory，后者在jdk的Executors也有实现。

# 创建NioEventLoopGroup

EventLoopGroup bossGroup = new NioEventLoopGroup(1);
EventLoopGroup workerGroup = new NioEventLoopGroup();

可以看到就是new一个实例出来，最终执行的是:

public NioEventLoopGroup(int nThreads, Executor executor, final SelectorProvider selectorProvider,
                            final SelectStrategyFactory selectStrategyFactory) {
    super(nThreads, executor, selectorProvider, selectStrategyFactory, RejectedExecutionHandlers.reject());
}

继续跟代码，这里可以看到用Object... args这种方式来接收多个参数：

protected MultithreadEventLoopGroup(int nThreads, Executor executor, Object... args) {
    super(nThreads == 0 ? DEFAULT_EVENT_LOOP_THREADS : nThreads, executor, args);
}

继续跟到父类，executor前面传的是null，看注释会用默认的实现：

/**
    * Create a new instance.
    *
    * @param nThreads          the number of threads that will be used by this instance.
    * @param executor          the Executor to use, or {@code null} if the default should be used.
    * @param args              arguments which will passed to each {@link #newChild(Executor, Object...)} call
    */
protected MultithreadEventExecutorGroup(int nThreads, Executor executor, Object... args) {
    this(nThreads, executor, DefaultEventExecutorChooserFactory.INSTANCE, args);
}

再往下跟，就是真正的创建实例的代码了：

/**
    * Create a new instance.
    *
    * @param nThreads          the number of threads that will be used by this instance.
    * @param executor          the Executor to use, or {@code null} if the default should be used.
    * @param chooserFactory    the {@link EventExecutorChooserFactory} to use.
    * @param args              arguments which will passed to each {@link #newChild(Executor, Object...)} call
    */
protected MultithreadEventExecutorGroup(int nThreads, Executor executor,
                                        EventExecutorChooserFactory chooserFactory, Object... args) {
    if (nThreads <= 0) {
        throw new IllegalArgumentException(String.format("nThreads: %d (expected: > 0)", nThreads));
    }

    if (executor == null) {
        executor = new ThreadPerTaskExecutor(newDefaultThreadFactory());
    }

    children = new EventExecutor[nThreads];

    for (int i = 0; i < nThreads; i ++) {
        boolean success = false;
        try {
            children[i] = newChild(executor, args);
            success = true;
        } catch (Exception e) {
            // TODO: Think about if this is a good exception type
            throw new IllegalStateException("failed to create a child event loop", e);
        } finally {
            if (!success) {
                for (int j = 0; j < i; j ++) {
                    children[j].shutdownGracefully();
                }

                for (int j = 0; j < i; j ++) {
                    EventExecutor e = children[j];
                    try {
                        while (!e.isTerminated()) {
                            e.awaitTermination(Integer.MAX_VALUE, TimeUnit.SECONDS);
                        }
                    } catch (InterruptedException interrupted) {
                        // Let the caller handle the interruption.
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }
        }
    }

    chooser = chooserFactory.newChooser(children);

    final FutureListener<Object> terminationListener = new FutureListener<Object>() {
        @Override
        public void operationComplete(Future<Object> future) throws Exception {
            if (terminatedChildren.incrementAndGet() == children.length) {
                terminationFuture.setSuccess(null);
            }
        }
    };

    for (EventExecutor e: children) {
        e.terminationFuture().addListener(terminationListener);
    }

    Set<EventExecutor> childrenSet = new LinkedHashSet<EventExecutor>(children.length);
    Collections.addAll(childrenSet, children);
    readonlyChildren = Collections.unmodifiableSet(childrenSet);
}

这段代码有许多值得学习借鉴的地方，比如创建多个线程时中途出现异常如何优雅的关闭所有线程。还有 这段代码chooser = chooserFactory.newChooser(children);如果跟进去的话，会看到netty利用位运算来替代取余运算来提升性能。

可以看到netty用了匿名类的语法（Java匿名类参考)）:

final FutureListener<Object> terminationListener = new FutureListener<Object>() {
    @Override
    public void operationComplete(Future<Object> future) throws Exception {
        if (terminatedChildren.incrementAndGet() == children.length) {
            terminationFuture.setSuccess(null);
        }
    }
};

Java 8之后可以用lambda简写：

final FutureListener<Object> terminationListener = (future) -> {
        if (terminatedChildren.incrementAndGet() == children.length) {
            terminationFuture.setSuccess(null);
        }
};

jdk future的用法参考

netty没用jdk的Future接口，而是自己定义了一个io.netty.util.concurrent.Future，其中有addListener方法：

for (EventExecutor e: children) {
    e.terminationFuture().addListener(terminationListener);
}

创建好FutureListener之后，加入到每一个事件线程中，等到线程结束后就会调FutureListener的operationComplete方法，我理解这是一种CallBack编程，也是观察者模式。这里其实有两层Future。

最后，children[i] = newChild(executor, args);执行的是NioEventLoopGroup的如下方法：

@Override
protected EventLoop newChild(Executor executor, Object... args) throws Exception {
    EventLoopTaskQueueFactory queueFactory = args.length == 4 ? (EventLoopTaskQueueFactory) args[3] : null;
    return new NioEventLoop(this, executor, (SelectorProvider) args[0],
        ((SelectStrategyFactory) args[1]).newSelectStrategy(), (RejectedExecutionHandler) args[2], queueFactory);
}
