# 原型模式

原型模式是23种设计模式之一，本身并不复杂，在我的博客中也有介绍原型模式。在netty中，ServerBootstrap就用到了原型模式，这里提取相关代码一窥究竟：

private ServerBootstrap(ServerBootstrap bootstrap) {
    super(bootstrap);
    childGroup = bootstrap.childGroup;
    childHandler = bootstrap.childHandler;
    childOptions.putAll(bootstrap.childOptions);
    childAttrs.putAll(bootstrap.childAttrs);
}

@Override
@SuppressWarnings("CloneDoesntCallSuperClone")
public ServerBootstrap clone() {
    return new ServerBootstrap(this);
}

以上代码的关键是把this作为参数传给构造函数，也就是把类的实例传给类的构造函数用于构造下一个实例

ServerBootstrap是AbstractBootstrap的子类，后者实现了Cloneable接口，super(bootstrap);调的是AbstractBootstrap的构造方法：

public abstract class AbstractBootstrap<B extends AbstractBootstrap<B, C>, C extends Channel> implements Cloneable {

    AbstractBootstrap(AbstractBootstrap<B, C> bootstrap) {
        group = bootstrap.group;
        channelFactory = bootstrap.channelFactory;
        handler = bootstrap.handler;
        localAddress = bootstrap.localAddress;
        options.putAll(bootstrap.options);
        attrs.putAll(bootstrap.attrs);
    }

    //省略其他代码
}

对比教科书上的原型模式，包括自己在github上的原型模式demo，可以清楚的看到netty的做法不一样，它没有调Object.clone()，而是利用this将字段拷贝了一份。

TIP

Cloneable是一个接口，但是并没有定义任何方法，它仅仅是用来标记可以合法的使用Object.clone()方法。如果没有实现Cloneable，却调用了Object.clone()，就会抛出CloneNotSupportedException。由于Cloneable并没有定义clone方法，实现它的类并不保证真的能够被clone。

Object.clone()方法可以返回Object的一份拷贝，一般来说以下三个表达式返回结果都为true:

  • x.clone() != x
  • x.clone().getClass() == x.getClass()
  • x.clone().equals(x)

但是这并不是绝对的要求。按惯例，通过调super.clone来返回Object，如果一个类和它所有的超类都遵守这个惯例，x.clone().getClass() == x.getClass()的结果就为true。

按惯例拷贝返回的Object应该独立于原来的Object，为了达到这种独立性，也许应该在返回结果前修改一个或多个字段。一般来讲，这意味着Object内部有可变的引用类型字段时，拷贝的时候要把这些字段也拷贝一份。当然，如果Object内部都是不可变的对象，就没必要拷贝对象内部的字段了。

Object.clone()方法首先会判断对象是否实现了Cloneable接口，没有的话会抛出CloneNotSupportedException。注意，所有的arrays都会实现Cloneable接口。接着创建对象，初始化所有的字段为被拷贝对象相应的字段值，引用类型的内容不会被拷贝。也就是说做的是浅拷贝，而不是深拷贝。

# 模板方法

EchoServerHandler继承的是ChannelInboundHandlerAdapter，但实际上在io.netty.example包下其他例子大部分都继承自SimpleChannelInboundHandler，它通过使用模板方法模式来简化了handler的编写。

对比模板方法模式中描述的类结构，netty的SimpleChannelInboundHandler是模板方法模式的抽象类，模板方法是channelRead，抽象方法是channelRead0，请看源码：

@Override
public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
    boolean release = true;
    try {
        if (acceptInboundMessage(msg)) {
            @SuppressWarnings("unchecked")
            I imsg = (I) msg;
            channelRead0(ctx, imsg);
        } else {
            release = false;
            ctx.fireChannelRead(msg);
        }
    } finally {
        if (autoRelease && release) {
            ReferenceCountUtil.release(msg);
        }
    }

protected abstract void channelRead0(ChannelHandlerContext ctx, I msg) throws Exception;

以这样的方式，上层自己实现handler时可以不必每次都调ReferenceCountUtil.release(msg);
