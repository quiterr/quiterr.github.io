# netty源码学习-3种IO、Reactor

  • netty对三种IO的支持
  • netty对Reactor的支持

3种IO指BIO,NIO,AIO，Reactor是一种请求处理的模式并且也有三种实现，网上讲这些的很多，我收藏的jasongj的博客就讲得非常好，这里不再重复介绍了。我们一般直接用netty的NIO和主从Reactor就行了。

# netty对三种IO的支持

还是看io.netty.example包下的EchoServer，其中有两行代码：

EventLoopGroup bossGroup = new NioEventLoopGroup(1);
EventLoopGroup workerGroup = new NioEventLoopGroup();

TIP

netty中group的含义就是jdk中的pool，EventLoopGroup就是处理事件的线程池。

很明显这就是NIO的用法了，那在netty中怎么用BIO、AIO呢？事实上在netty里和BIO对应的是OIO(old IO)。而AIO在netty中曾经有过对应的实现，后来删除了，原因是linux系统对AIO的支持不够好。

OIO的用法如下:

EventLoopGroup bossGroup = new OioEventLoopGroup();
EventLoopGroup workerGroup = new OioEventLoopGroup();

不过OIO相关的实现类都已经被标记为@Deprecated，并不推荐使用了。

下表是netty对三种IO的支持情况：

BIO/OIO （已废弃） Common NIO Linux NIO macOS/BSD NIO AIO（已删除）
OioEventLoopGroup NioEventLoopGroup EpollEventLoopGroup KqueueEventLoopGroup AioEventLoopGroup

# netty对Reactor的支持

经典Reactor

EventLoopGroup group = new NioEventLoopGroup(1);

多线程Reactor（不指定线程数，默认就会按CPU的核数来配置线程数）

EventLoopGroup group = new NioEventLoopGroup();

主从Reactor

EventLoopGroup bossGroup = new NioEventLoopGroup(1);
EventLoopGroup workerGroup = new NioEventLoopGroup();
