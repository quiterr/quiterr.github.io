# netty源码学习-反射和工厂

    在netty源码学习-链式编程中，贴过一段众所周知的代码：



     














    ServerBootstrap b = new ServerBootstrap();
b.group(bossGroup, workerGroup)
        .channel(NioServerSocketChannel.class)
        .option(ChannelOption.SO_BACKLOG, 100)
        .handler(new LoggingHandler(LogLevel.INFO))
        .childHandler(new ChannelInitializer<SocketChannel>() {
            @Override
            public void initChannel(SocketChannel ch) throws Exception {
                ChannelPipeline p = ch.pipeline();
                if (sslCtx != null) {
                    p.addLast(sslCtx.newHandler(ch.alloc()));
                }
                //p.addLast(new LoggingHandler(LogLevel.INFO));
                p.addLast(serverHandler);
            }
        });

    第三行高亮的代码，channel方法入参是NioServerSocketChannel.class，channel方法的定义如下：

    public B channel(Class<? extends C> channelClass) {
    return channelFactory(new ReflectiveChannelFactory<C>(
            ObjectUtil.checkNotNull(channelClass, "channelClass")
    ));
}

    可见，要用类名来做函数参数，接收的时候得写成Class<? extends C> channelClass，没有泛型的时候，写成Class clazz。

    这里netty用类名来做函数参数，我想是因为channel的种类真的很多，包括Server端的和Client端的，例如在io.netty.channel.socket.nio包下就有: NioDatagramChannel,NioServerSocketChannel和NioSocketChannel。

    在io.netty.channel.epoll和io.netty.channel.kqueue包下也还有多个channel的实现。

    接着就是调channelFactory方法来创建channel实例，channelFactory方法的参数我第一眼看有点懵，因为平常没有这么写过代码：

    new ReflectiveChannelFactory<C>(
    ObjectUtil.checkNotNull(channelClass, "channelClass")
)

    用一个常用的代码来类比就清晰些了：

    ArrayList<Integer> list = new ArrayList<>();

    也就是说尖括号<>里边是泛型的具体类型，()是构造函数的参数。这里给ReflectiveChannelFactory的构造函数传的参数是：

    ObjectUtil.checkNotNull(channelClass, "channelClass")

    这是比较容易迷惑的地方，初看还以为传了一个方法，再看才发现ObjectUtil.checkNotNull返回值就是channelClass，也就是NioServerSocketChannel.class，这就和ReflectiveChannelFactory的构造函数的参数匹配上了：

    public class ReflectiveChannelFactory<T extends Channel> implements ChannelFactory<T> {
    // 省略了除构造函数以外的代码
    public ReflectiveChannelFactory(Class<? extends T> clazz) {
        ObjectUtil.checkNotNull(clazz, "clazz");
        try {
            this.constructor = clazz.getConstructor();
        } catch (NoSuchMethodException e) {
            throw new IllegalArgumentException("Class " + StringUtil.simpleClassName(clazz) +
                    " does not have a public non-arg constructor", e);
        }
    }
}

    ReflectiveChannelFactory看名字就知道这是一个用反射来生成对象的工厂，具体做的时候是用反射来获取无参数的构造函数。

    channel方法就是负责把ReflectiveChannelFactory这个工厂创建好，并没有创建Channel实例。

    由于只生产Channel这一种产品，这里应该是用的工厂方法模式。

    搜索代码可知只有ReflectiveChannelFactory这一个工厂，那是否可以认为这里是简单工厂模式？毕竟在创建型设计模式中讲过：“只生产一种产品，只有一个工厂的是简单工厂模式”。

    理论指导实践，但实践也有优化的地方，这里应该是用反射的方式优化了工厂方法模式，只用一个工厂就实现了多个不同的Channel的生产。还有代理模式也是一样，理论上可能只讲到了静态代理，实际更多的是动态代理。理论没有错，只是实践当中又在不断进化。
